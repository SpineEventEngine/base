Environment

@SPI()
public final class Environment implements Logging

Provides information about the environment (current platform used, etc.).

Detecting the type of the environment

It is possible to obtain the type of the current environment, or to check whether current environment type matches another type. 


public final class Application {

    private final EmailSender sender;

    private Application() {
        Environment environment = Environment.instance();
        if (environment.is(Tests.class)) {
            // Do not send out emails if in tests.
            this.sender = new MockEmailSender();
        } else {
            this.sender = EmailSender.withConfig("email_gateway.yml");
        }
        //...
    }
}

The following standard types are always available for checking with the current environment:

  • Tests is detected if the current call stack has a reference to a unit testing framework.
  • DefaultMode is set in all other cases.

The framework users may define their custom settings depending on the current environment type and then register them for further detection. Custom types are evaluated in the order reverse to registration. That is, last registered type would be checked first and so on. Please see CustomEnvironmentType for details.

When environment changes

When calculating the current typeEnvironment finds the first enabledEnvironmentType, and then remembers it as the currently active. Subsequent calls will not cause re-evaluation of the enabled.

If later the current type becomes logically inactive, Environment needs to be updated by one of the following approaches:

  • Setting a new environment type, directly by calling setTo.
  • Calling the reset method (which would drop the currently selected type) and re-registering environment types. 

    Environment environment = Environment.instance();
    environment.register(AwsLambda.class);
    assertThat(environment.is(AwsLambda.class)).isTrue();

    System.clearProperty(AwsLambda.AWS_ENV_VARIABLE);

    // Even though `AwsLambda` is not active, we have remembered the value, and
    // `is(AwsLambda.class)` is `true`.
    assertThat(environment.is(AwsLambda.class)).isTrue();

    environment.reset();
    environment.register(AwsLambda.class);

    // When `reset()` is called, the stored value is cleared.
    assertThat(environment.is(AwsLambda.class)).isFalse();
