CodeGenerator

public abstract class CodeGenerator

An abstract base for the Protobuf to Java code generator.

A generator consumes a DescriptorProto for each message type and optionally generates some Java code in response to it regarding its file.

Each message type is processed separately. As the result of processing, a generator may produce instances of CodeGeneratorResponse.File.

The CodeGeneratorResponse.File has three fields: name, insertionPoint, and content.

The name field represents the name of the file to generate. The name is relative to the output directory and should not contain ./ or ../ prefixes.

The content field represents the code snippet to write into the file. This field is required.

To make the protoc generate a new file from the scratch, the the generator should produce CodeGeneratorResponse.File instance with the name and content fields. The insertionPoint field is omitted in this case.

To extend an existing protoc plugin (e.g. built-in java plugin), use insertionPoint field. The value of the field must correspond to an existing insertion point declared by the extended plugin. The insertion points are declared in the generated code as follows: @@protoc_insertion_point(NAME), where NAME is value to set into the field.

If the insertionPoint field is present, the name field must also be present. The content field contains the value to insert into the insertion point is this case.
