public final void mark(PatternSelector patternSelector, ClassName interfaceName)

Configures an interface generation for messages declared in files matching a given pattern.

Sample usage is as follows: 

mark messages().inFiles(suffix: "events.proto"), asType("my.custom.EventMessage")
Content copied to clipboard

The statement in the example above configures all message types declared in a file which name ends with events.proto to implement the my.custom.EventMessage. It is expected that my.custom.EventMessage is an interface defined by the user.

Note that only the top-level messages declarations are affected by this configuration. Nested messages defined in the same file do not implement the interface.

Caution. In order for the framework components to function properly, one should not entirely override the default message interfaces. Instead, provide a custom interface which extends the standard one. Example of a safe way to override standard interfaces: 

// In my/custom/EventMessage.java:

package my.custom;

public interface EventMessage extends io.spine.base.EventMessage {
// ...
}

// In build.gradle:

// ...

modelCompiler {
    interfaces {
        mark messages().inFiles(suffix: "events.proto"), asType("my.custom.EventMessage")
    }
}

In the example above, my.custom.EventMessage extends io.spine.base.EventMessage and thus it is safe to mark all events with this interface instead of the default one.

public final void mark(UuidMessage uuidMessage, ClassName interfaceName)

Configures an interface generation for messages with a single string field called uuid.

This method functions similarly to the mark except for several differences:

  • the file in which the message type is defined does not matter;
  • nested definitions are affected as well as top-level ones.

Sample usage is as follows: 

mark messages().uuid(), asType("my.custom.Identifier")

public final void mark(EntityState entityState, ClassName interfaceName)

Configures an interface generation for messages that represent an entity state.

All messages marked with (entity) option and with a valid (kind) will be marked with the given interface name.

Sample usage is as follows: 

mark messages().entityState(), asType("my.custom.EntityState")

Note that it is required for the provided interface to extend the io.spine.base.EntityState interface, otherwise the inner Spine routines will work incorrectly.
