Generates the Validator class for the associated message type.

The Validator class is supposed to be nested inside the message class. It is declared private and static.

The name of the class is most often Validator. However, when a name clash occurs, the name is padded with the $ (dollar sign). For instance if the message itself is called Validator, the validator class will be called Validator$. Since the class is private, this naming is not exposed to the outer scope.

The only method of the class is: 

private staticList<ConstraintViolation> validate(MyMsg msg) { ... }

In this example, MyMsg is the type of the validated message.

The class is marked with the Generated annotation so that static code analysis can ignore it.
